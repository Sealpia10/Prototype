프로필 편집

같은 PC에서 LM Studio를 쓰는 경우가 가장 간단합니다: LM Studio → Developer(</>) 탭 → "Start Server"를 켜면 http://localhost:1234에 OpenAI 호환 API가 열립니다. 이 파일(Main.html)을 같은 PC에서 로컬 파일로 열면 위 주소를 그대로 입력해 바로 연결됩니다.
다른 PC/외부망에서 접속하려면 학교망은 대부분 443(HTTPS)만 허용하므로 Cloudflare Tunnel · ngrok으로 https 주소를 만들어 입력하세요. (http://나 :1234·:11434 직접 접속은 차단될 수 있음)
LM Studio · Ollama · oobabooga · vLLM 등 OpenAI 호환(/v1/chat/completions) 서버를 모두 지원합니다.

세부 조정

프로필·키는 이 기기에 저장되며 체험자 리셋으로 지워지지 않습니다. 공용 PC이므로 행사 후에는 프로필을 삭제해 키를 정리하세요.
※ ChatGPT 공식 API는 브라우저 직접 호출(CORS)이 막혀 있어, 실패 시 오류 로그에 기록됩니다. 그 경우 로컬 LLM(OpenAI 호환) 방식 사용을 권장합니다.

멀티 컴퓨터 모드

A 컴퓨터 1대가 LM Studio(로드한 모델)로 5개 페르소나를 전부 연산하고, 컴퓨터 1~5는 각자 담당 페르소나의 대화 요청을 A에 송신한 뒤 자기 몫의 답인지 검증해 수신합니다.

A 컴퓨터(호스트) 준비 ① LM Studio에서 원하는 모델을 로드 (gemma·llama·qwen·mistral·exaone 등 어떤 모델이든 지원 — 모델명이 정확하지 않아도 로드된 모델로 자동 보정됩니다) → ② Developer(</>) 탭 → Start Server → "Serve on Local Network"를 켜기 (그래야 다른 컴퓨터가 http://A의IP:1234 로 접속 가능) → ③ CORS 허용(기본값) 확인. A의 IP는 Windows에서 ipconfig, macOS에서 ifconfig로 확인합니다.
컴퓨터 1~5(클라이언트) 이 파일(Main.html)을 로컬 파일로 열고, 역할을 클라이언트로 → A의 주소와 이 컴퓨터 번호를 저장하세요. 이후 대화를 보내면 자동으로 A에 송신 → A의 gemma가 담당 페르소나로 연산 → 이 컴퓨터가 자기 몫의 답을 수신해 표시합니다. (사이트를 https로 열면 http://A주소가 브라우저에 차단되니 반드시 로컬 파일로 여세요)
다른 망(외부망)에서 접속하는 경우 A에서 ngrok http 1234 또는 cloudflared tunnel --url http://localhost:1234를 실행해 나온 https 주소를 위 서버 주소 칸에 입력하세요. ngrok 무료 플랜의 브라우저 경고 페이지는 자동으로 우회하며, https 터널 주소는 사이트를 https로 열어도 차단되지 않습니다. 연결이 안 되면 ① 터널 프로그램이 켜져 있는지 ② 주소를 새로 발급받았는지(무료 플랜은 재시작 시 주소가 바뀜) ③ "A 서버 연결 확인" 버튼의 오류 메시지를 확인하세요.
여러 컴퓨터가 동시에 질문하면 A가 순서대로 연산하므로 답이 조금 늦어질 수 있으며, 이때는 자동으로 재시도합니다. 멀티PC 모드가 켜져 있으면 "AI 연동" 탭의 프로필 대신 이 설정이 사용됩니다.

비속어 검열

사용자가 입력해 전송하는 순간, 등록된 검열어를 ○로 마스킹합니다. 화면 표시와 AI에게 전달되는 문장 모두 마스킹된 상태가 됩니다.

JSON은 ["단어", …] 배열, {"words": […]}, 중첩 객체 등 문자열이 담긴 어떤 구조든 자동으로 훑어 등록합니다. 목록은 이 기기에 저장되며 체험자 리셋으로 지워지지 않습니다.
서버 측(A 컴퓨터) 일괄 검열이 필요하면 middleware/filter_proxy.py를 함께 사용하세요 — 같은 txt/csv/json 파일을 공유할 수 있습니다.
