- `showStoryChoices`가 선택지의 `hideIfFlag`/`needFlag`를 지원 —
  플래그로 선택지를 노출/숨김 처리, 남은 선택지가 모두 소진되면 다음 노드로 자동 진행
  (이미 감상한 코스프레 의상을 목록에서 제거하는 데 사용)
- **모든 스토리 모드 대사·내레이션·선택지의 주인공 지칭을 체험자 이름으로 치환**

  플레이스홀더를 입력한 이름으로 바꾸고, 뒤따르는 조사(은/는·이/가·을/를·과/와·(으)로·아/야)를
  이름의 받침 유무에 맞춰 자동 보정. 타자기 즉시완료·선택지 라벨에도 동일 적용되며
  미코뿐 아니라 츠키·타로·마후유·유메 루트 전체에 반영됨

### 변경 (자유모드 검열)
- **검열은 화면 표시(UI)에만 적용, LLM에는 원본 비속어를 그대로 전달**
  — 검열어가 사용자 말풍선과 캐릭터 답변 표시에서는 `○`로 마스킹되지만, LM Studio 등
    LLM 모델에는 원본 문장이 그대로 전송된다. 이를 통해 사용자가 직접 추가한 검열어에도
    모델이 상황에 맞는 **다양한 부정 반응**을 직접 생성한다.
- **검열어 입력 시 호감도는 반드시 감소** — 모델이 0/양수 델타를 반환해도 하한 −5를 강제
  (`Math.min(delta, -5)`). 반복 시 음수가 되어 기존 강제종료로 이어짐.
- **AI 연결 실패 시 폴백** — 모델 호출이 실패하면 캐릭터별 결정론적 부정 반응
  (`PROFANITY_REACTIONS`, 화남/서운함)과 보장된 감소(검열어 1건당 −5, 최대 −30)로 대체해
  오프라인에서도 부정 반응·호감도 감소를 보장
